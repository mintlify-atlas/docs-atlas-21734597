# Quick start

This guide will walk you through creating a complete Node.js service with environment validation, async setup, and graceful shutdown.

## Basic entrypoint

Let's start with the simplest possible entrypoint:

```typescript
import { entry } from "entrykit";

entry({
  name: "my-service",
  main: () => {
    console.log("Service started.");
    return () => console.log("Shutting down.");
  },
});
```

The function returned from `main` is automatically registered as a SIGTERM and SIGINT handler for graceful shutdown.

<Tip>
Try running this with `node` and pressing Ctrl+C - you'll see the shutdown message.
</Tip>

## Environment validation

Production services need environment variables. EntryKit validates them at startup using ArkType:

<Steps>
  <Step title="Import the type function">
    EntryKit re-exports `type` from ArkType for convenience:
    
    ```typescript
    import { entry, type } from "entrykit";
    ```
  </Step>
  
  <Step title="Define your environment schema">
    Use ArkType's schema syntax to define required variables:
    
    ```typescript
    entry({
      name: "api-server",
      env: type({
        DATABASE_URL: "string",
        PORT: "string",
      }),
      main: ({ env }) => {
        // env.DATABASE_URL and env.PORT are validated and typed
        console.log(`Connecting to ${env.DATABASE_URL}`);
      },
    });
    ```
  </Step>
  
  <Step title="Access validated variables">
    The `env` parameter in `main` is fully typed and validated. If any required variable is missing or invalid, ArkType throws before `main` runs.
  </Step>
</Steps>

<Note>
ArkType validates at runtime AND provides TypeScript types. You get both compile-time and runtime safety.
</Note>

## Complete example with async setup

Real services often need async initialization. Here's a complete example with database connections and an HTTP server:

```typescript
import { entry, type } from "entrykit";
import { createServer } from "http";

// Mock database connection for demonstration
const connectToDatabase = async (url: string) => {
  console.log(`Connecting to database: ${url}`);
  await new Promise(resolve => setTimeout(resolve, 1000));
  return {
    query: (sql: string) => console.log(`Executing: ${sql}`),
    disconnect: async () => {
      console.log("Database disconnected");
    },
  };
};

entry({
  name: "api-server",
  env: type({
    DATABASE_URL: "string",
    PORT: "string",
  }),
  setup: async ({ env }) => {
    // Async setup phase - runs before main
    const db = await connectToDatabase(env.DATABASE_URL);
    return { db };
  },
  main: ({ env, extras }) => {
    // extras.db is the resolved value from setup
    const server = createServer((req, res) => {
      extras.db.query("SELECT * FROM users");
      res.writeHead(200, { "Content-Type": "text/plain" });
      res.end("Hello from EntryKit!");
    });

    server.listen(env.PORT, () => {
      console.log(`Server listening on port ${env.PORT}`);
    });

    // Return cleanup function for graceful shutdown
    return async () => {
      console.log("Shutting down server...");
      server.close();
      await extras.db.disconnect();
      console.log("Cleanup complete");
    };
  },
});
```

## Understanding the lifecycle

When you run the example above, EntryKit executes in this order:

<Steps>
  <Step title="Environment validation">
    EntryKit validates `DATABASE_URL` and `PORT` exist in `process.env`. If validation fails, the process exits immediately with an ArkType error.
  </Step>
  
  <Step title="Setup execution">
    The `setup` function runs asynchronously. The database connection is established and returned.
  </Step>
  
  <Step title="Main execution">
    The `main` function receives validated `env` and `extras` (containing `db`). The HTTP server starts listening.
  </Step>
  
  <Step title="Shutdown registration">
    Since `main` returned a cleanup function, EntryKit registers it for SIGTERM and SIGINT signals.
  </Step>
</Steps>

## Running your service

Create a `.env` file or export environment variables:

```bash
export DATABASE_URL="postgresql://localhost:5432/mydb"
export PORT="3000"
```

Then run your service:

<CodeGroup>

```bash bun
bun run index.ts
```

```bash node
node --import tsx index.ts
```

```bash tsx
tsx index.ts
```

</CodeGroup>

Press Ctrl+C to trigger graceful shutdown. You'll see the cleanup function execute.

## Advanced ArkType patterns

ArkType supports sophisticated validation. Here are some useful patterns:

```typescript
import { entry, type } from "entrykit";

entry({
  name: "advanced-service",
  env: type({
    // String with format validation
    DATABASE_URL: "string",
    
    // Number validation with parsing
    PORT: "string.numeric.parse",
    
    // Optional with default
    LOG_LEVEL: "'debug' | 'info' | 'warn' | 'error' = 'info'",
    
    // Boolean parsing
    ENABLE_METRICS: "string.numeric.parse > boolean",
  }),
  main: ({ env }) => {
    console.log(env.PORT); // Type: number
    console.log(env.LOG_LEVEL); // Type: "debug" | "info" | "warn" | "error"
  },
});
```

<Tip>
Explore [ArkType's documentation](https://arktype.io) for more validation patterns like email validation, URL parsing, and custom validators.
</Tip>
